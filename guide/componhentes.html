# Compoñentes

Un compoñente é unha peza de software reutilizable. É dicir, consiste en programar algo que vai ser reutilizado noutros proxectos.

En terminoloxía Java, un compoñente software é unha clase creada para ser reutilizada. Defínese polo seu estado, que se almacena nun conxunto de propiedades, as cales poden ser modificadas para adaptar o compoñente ao programa no que se insira, e un comportamento, que se define polos eventos aos que responde e os métodos que executa ante ditos eventos.

Programar un compoñente dende cero é bastante complicado, pois hai que programar dende como se renderiza en pantalla ata como responde aos eventos, o que significa moitas liñas de código. Realmente o que nós imos facer é coller como base un compoñente de Swing e estendelo a través do mecanismo de herdanza creando así un compoñente personalizado.

Un subconxunto dos atributos e os métodos forman a interface do compoñente. Os compoñentes Swing utilizados ata agora nas interfaces gráficas elaboradas son elementos reutilizables que poden ser manexados por unha ferramenta de desenvolvemento visual. Por exemplo, pode modificarse o seu tamaño e cor para adaptalo á interface e estes cambios permanecen almacenados no proxecto. Tamén teñen unha interface formada por un conxunto de métodos e propiedades accesibles dende a paleta de propiedades. Ademais, teñen a capacidade de responder a eventos.
